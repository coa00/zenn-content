## はじめに

Web サービスの開発に関わっていると、リリースのたびに「ログインできるか」「主要な画面が問題なく表示されるか」を一通り確認する場面があります。エンジニアでなくても、ディレクターやデザイナーとして、自分の関わるサービスが正しく動くか気になり、手元で確認されている方は多いのではないでしょうか。

この記事では、その確認作業を Claude Code の Chrome 拡張（Claude in Chrome）を使い、日本語の指示だけで進める方法をご紹介します。コードを書く必要はありません。実際に試した手順と、使ってみて分かったことを順を追ってまとめます。

コードは書かないけれど動作確認には関わる、デザイナー・ディレクターの方を念頭に書いています。

## Claude in Chrome とは

Claude in Chrome は、Claude を提供している Anthropic 公式の Chrome 拡張機能です。これを導入すると、Claude Code から普段お使いの Chrome を操作できるようになります。

特徴は、**新しい専用ブラウザを立ち上げるのではなく、いつもの Chrome をそのまま使う**点です。すでにログイン済みのサイトであれば、その状態から続けて確認できます。

:::message
利用には、Claude の有料プラン（Pro / Max / Team / Enterprise のいずれか）と、お使いのパソコンに Claude Code が導入されていることが必要です。対応ブラウザは Chrome または Microsoft Edge です。2026 年 6 月時点ではベータ版（試験提供）として提供されています。
:::

## セットアップ

前提として、本記事は Claude Code がすでに動作している状態を想定しています。まだ導入されていない場合は、公式の手順に沿ってインストールしてください。一度 Claude Code が動いていれば、この先で難しいコマンドを覚える必要はありません。最初に一度だけ Chrome との接続設定を行えば、あとは日本語で指示するだけです。

手順は次のとおりです。

1. Chrome ウェブストアで「Claude in Chrome」を検索し、拡張機能を追加します。
2. Claude Code を起動し、`/chrome` と入力して「Enabled by default（既定で ON）」を選びます。これで Chrome と接続されます。一度設定すれば、次回以降は Claude Code を起動するだけで自動的に接続されます。
3. 確認したいサイトを操作対象として許可します。どのサイトを操作してよいかは拡張機能の設定で指定し、許可していないサイトは操作されません。

なお、拡張機能を入れた直後に Claude Code 側で「拡張が見つからない」と表示される場合があります。これは初回のみ設定ファイルの再読み込みが必要なためで、Chrome を再起動すると解消します。同じ状況になった際は、まず Chrome の再起動をお試しください。

## 基本的な使い方

接続後は、Claude Code に日本語で依頼するだけです。特別な記号やコマンドは必要ありません。

たとえば、次のように依頼します。

> このページを開いて、ログインして、トップ画面が問題なく表示されるか確認してください。

すると Claude が Chrome を操作し、結果を文章で報告します。

> ログインに成功し、ダッシュボードが表示されました。一覧に項目が表示されており、エラーは出ていません。

「どこまで確認できたか」「どこで表示されなかったか」を、人に報告してもらうように伝えてくれるため、状況が把握しやすいのが利点です。

## 実際に試してみる（ログインの確認）

開発中の Web アプリで、ログインまわりを確認してみました。準備を除けば、所要時間は数分です。

最初に一点つまずいたのが、対象の取り違えです。「ローカルで動いているものを見て」とだけ依頼したところ、別のツールの画面が開いてしまいました。手元で複数のものが動いていたため、対象が定まらなかったのが原因です。**URL を明確に指定し直したところ**、目的のアプリが正しく開きました。依頼内容が曖昧だと結果も曖昧になる点は、人に作業をお願いするときと同じだと感じました。

ログインの確認では、**パスワードの入力は自分で行う**点が重要です。Claude はログイン画面に到達すると自動的に停止し、入力を利用者に委ねます（パスワードを自動で入力しない設計になっています）。こちらでメールアドレスとパスワードを入力してサインインすると、その後の確認は Claude が引き継ぎ、ダッシュボードに遷移したことを確認してくれました。ログイン → サインアウト → 再ログインの流れを、**操作の大半は Claude、ログインのみ自分**という分担で進められました。

最後に、一覧から詳細画面を開き、レイアウトが崩れていないかを確認しました。各項目が正しく表示され、画面の裏側で発生するエラー（コンソールエラー）もありませんでした。「エラーは見つかりませんでした」という報告で、確認は完了です。

手作業でも数分で終わる確認ではありますが、これを毎回繰り返さずに済むと考えると、意味合いは変わってきます。

## 複数画面の確認・スクリーンショット・登録操作

ログインの確認に続けて、もう少し踏み込んで試しました。

便利だったのは、**複数の画面をまとめて確認してもらう**使い方です。「ログイン後、主要な画面を順番に開き、表示の崩れ・データの有無・コンソールエラーの有無をまとめて確認してください」と一度依頼すると、20 画面ほどを順に開き、「この画面は表示できました」「これも問題ありません」「エラーはありませんでした」と一覧で報告してくれます。

**スクリーンショットの取得も可能です。** 「この画面のスクリーンショットを撮ってください」と依頼すると画像を残せるため、気になる箇所の記録や、報告への添付に利用できます。一連の操作を GIF 動画として記録することもでき、手順を共有したい場面で役立ちます。

注意が必要だったのは、**表示に時間がかかる画面**です。開いた直後は内容が表示されておらず、Claude が「空白でした」と報告することがあります。これは描画が間に合っていないだけのため、「少し待ってから確認してください」と依頼し直すと、正しく内容が表示されました。

**登録（書き込み）の操作も確認しました。** フォームに値を入力してもらい、**送信ボタンを押す前に、入力内容を読み上げて確認**してもらいます。「この内容で登録してよいか」を一度確認したうえで送信する、という進め方です。一方で、**削除のように取り消せない操作の最終ボタンは、自分で押す**ようにしました（Claude は恒久的な削除のような操作を自動では実行しません）。人とツールで役割を分けておくと安心です。

また、権限のない画面を開いた際に「アクセス権限がありません」と表示される場面もありました。これは不具合ではなく、**アクセス制御が正しく機能している状態**です。Claude も「アクセスできませんでした」とそのまま報告するため、こうした権限の確認にも利用できます。

## 使う際の注意点

実際に使って気づいた点を、いくつか挙げておきます。

- **サイトの許可を忘れると操作されません。** 操作が始まらない場合は、まず拡張機能の設定で対象サイトを許可しているか確認してください。
- **確認ダイアログで処理が止まることがあります。** 「本当に削除しますか」のようなポップアップが表示されると、ブラウザがその状態で停止することがあります。表示された場合は手動で閉じてから、続きを依頼してください。削除のような操作は、はじめのうちは避けておくと安全です。
- **うまくいかないときは、依頼を具体的にします。** 「どの画面の、どの操作を、どう確認してほしいか」を明確に伝えると、動作が安定します。

なお、ログインや CAPTCHA（「私はロボットではありません」の確認）が表示されると、Claude は自動的に停止し、操作を利用者に委ねます。自動で突破しようとはしないため、この点は安心して利用できます。

## 試した項目と所要時間

今回試した項目を整理すると、次のとおりです（拡張機能の導入や接続設定の時間は除いています）。

| 試した項目 | 内容 | 目安の時間 |
|---|---|---|
| ログインの確認 | サインアウト → 再ログインを含む。ログインの入力は自分で行う | 約 5 分 |
| 主要画面の表示確認 | 約 20 画面を順に開き、表示・データの有無・コンソールエラーを確認 | 約 10〜15 分 |
| 詳細画面のレイアウト確認 | 一覧から詳細画面を開き、崩れがないかを確認 | 数分 |
| スクリーンショットの取得 | 画面の記録・報告への添付 | 都度 |
| 登録（書き込み）の確認 | フォーム入力 → 内容確認 → 送信 | 約 5 分 |
| 削除の確認 | 取り消せない操作の最終ボタンは自分で実行 | 数分 |
| 権限の確認 | 権限のない画面で「アクセス権限がありません」と表示されることを確認 | 数分 |

ひととおりで、30 分ほどでした。手作業で同じ範囲を確認する場合、画面数が多いほど時間がかかりますし、何よりリリースのたびに繰り返す負担がなくなります。

## まとめ

動作確認の作業は、これまで指示をプログラミング言語で書く必要があり、エンジニア以外には手を出しにくいものでした。Claude Code の Chrome 拡張を使うと、その指示を日本語で行えるようになり、コードを書かずに確認を進められます。

行うことは「拡張機能を入れる」「日本語で依頼する」の二つだけです。リリースのたびに同じ画面を手で確認している場合は、まずは一つの画面から「これが正しく表示されるか確認してください」と依頼してみてはいかがでしょうか。デザイナーやディレクターの方でも、無理なく取り入れられる方法だと思います。

もし試された際は、使い勝手や、より良い使い方があればぜひ共有いただけますと幸いです。
